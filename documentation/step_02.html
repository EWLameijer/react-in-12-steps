Step 2: Transform the template app into a very simple app of your own

You now have a running app. But you likely want to make something very different from a counter app! Let's go over the files and modify them as needed, along the way I'll discuss some other useful things to know about React.

Index.html

By convention, most webpages are called "index.html", since that is the page automatically loaded by the server when it receives a HTTP request. If I would have an org.wubbo server, where the actual server ran at C:\website, a browser going to org.wubbo/lessons would actually load C:\website\lessons\index.html. Likely the name comes from the old days of the internet, where you wanted an index to explore which files from the server's directory you actually needed. Anyway, in all projects check for the location of the index.html, as that is usually the starting point for all JavaScript applications.

But what should we look for at index.html? Well, it looks like

<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vite + React</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>

Most here is standard. I remove the icon link (don't need the vite icon), and change the title into something like ReactTest230723

I wish to point out two further items.

First, the div with id=root. This is where React will later insert the actual components.

Second, the inclusion of the main.jsx script. This is the place where the React code is loaded. So let's check out the main.jsx! (after taking a final look at the new index.html) 

  <!doctype html>
  <html lang="en">
    <head>
      <meta charset="UTF-8" />
      <meta name="viewport" content="width=device-width, initial-scale=1.0" />
      <title>ReactTest230723</title>
    </head>
    <body>
      <div id="root"></div>
      <script type="module" src="/src/main.jsx"></script>
    </body>
  </html>

main.jsx

main.jsx is quite short:

    import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'

ReactDOM.createRoot(document.getElementById('root')).render(
  
    
  ,
)

First, it imports React and ReactDOM, which are the libraries necessary for using the basics of React (JSX, state), and for displaying its output on web pages, respectively.

Then, it imports an App and an index.css

index.css is undoubtedly the stylesheet for index.html, which is apparently imported via the script instead of directly in index.html.

Note that in contrast to the first two imports, the last two imports start with './'. This means that they are on the 'ordinary' path in the src directory, instead of in the node_modules directory.

What the App does? We'll see soon. But basically, App is a React component, which are generally imported with statements like these.

Then there is some 'React magic'. The ReactDOM.createRoot fills the root div in index.html with the contents of the App component. Again, notice some things:

  1. The render method does not seem to take an ordinary string, array, object or even regex, but some literal that looks like HTML. This is a special notation of React, called JSX (JavaScript XML)
